# Report Outline

Use this file to draft your report in your own words before converting it to
PDF. The assessment says the report should stay below 3000 words.

## 1. Introduction

Write about:

- the organization or context you selected
- the problem the software solves
- the purpose and scope of the program

Starter questions:

- Who will use the system?
- What tasks does it make easier?
- What is included and not included in the system?

## 2. System Design

Write about:

- the main classes
- the attributes and methods in each class
- how the classes work together
- how files are used to store data

Optional diagram:

```text
Library
  |-- manages many Book objects
  |-- manages many Member objects
  |-- manages many LoanRecord objects
```

## 3. Implementation Overview

Write about:

- where classes and objects are used
- where loops and conditionals are used
- where file input/output is used
- where exception handling is used
- how the program is split into modules
- how comments and docstrings make the code readable

## 4. Testing and Demonstration

Include examples from your own program run:

- normal input and output
- invalid input and error handling
- borrowing and returning a book
- searching for existing and missing records

You can paste short terminal examples here.

## 5. Reflection

Write about:

- challenges you faced
- what you learned
- what you would improve in the future

Possible improvements:

- add login for librarians
- add due dates and overdue fines
- add a graphical user interface
- use a database instead of CSV files

## References

Use Harvard referencing if you use sources such as Python documentation,
tutorials, or textbook material.
